# Account and Security Management

These documents guide **QuickBite** users (the fictitious food delivery app) on creating, managing, and securing their accounts. QuickBite prioritizes user privacy and security with features like phone/email verification, strong password policies, two-factor authentication (2FA), and quick recovery options. All data is handled per our [Privacy Policy](#) and encrypted in transit/storage.

Access these features via Profile > Account & Security in the app (available on iOS and Android).

## 1. Account Setup and Recovery Guide

**Creating a New Account**
QuickBite requires a verified phone number or email for signup to prevent fraud and enable easy recovery.

**Step-by-Step Signup**:

1. Open QuickBite app > Tap "Sign Up".
2. Enter your phone number (recommended for OTP delivery) or email.
3. Receive a 6-digit OTP (one-time password) via SMS/email.
4. Enter OTP to verify.
5. Create a strong password: Minimum 8 characters, including uppercase, lowercase, number, and special character (e.g., !@#$%).
6. Add optional details: Name, delivery address (use current location in Pitampura, Delhi for accuracy), and profile photo.
7. Accept Terms of Service and Privacy Policy.
8. You're in! Enable 2FA immediately for extra protection (see below).

**Account Recovery (Forgot Password or Locked Out)**
If you forget your password or can't log in:

1. On login screen > Tap "Forgot Password?" or "Trouble Signing In?".
2. Enter registered phone/email.
3. Receive OTP via SMS/email (or use backup method if set).
4. Verify OTP > Set new password.
5. If 2FA is enabled and you can't access your authenticator/app:
   - Use one of your 10 backup recovery codes (generated during 2FA setup—save them securely!).
   - Or contact support via in-app chat/email (support@quickbite.com) with identity proof (e.g., order history screenshot, registered phone).
6. Once recovered, review recent activity and change password immediately.

**Tips**:

- Use a unique password not reused elsewhere.
- Add a recovery email/phone in Profile > Account Settings > Recovery Options.
- For lost phone: Log in from another device first, then disable old 2FA and set up new.

## 2. Profile Management Guide

Keep your profile up-to-date for seamless ordering and security.

**Key Profile Sections**:

- **Personal Info**: Update name, phone/email, date of birth (for age-restricted items).
- **Addresses**: Add/edit multiple delivery addresses (e.g., home in Pitampura, office). Set default, add notes (e.g., "Gate code 4567").
- **Payment Methods**: View/add/remove cards, UPI, wallets (see Payment Guide). QuickBite uses tokenized storage—no full card details saved.
- **Preferences**: Dietary tags (veg/non-veg), favorite restaurants/dishes, notification settings.
- **Order History**: View past orders, reorder favorites, download invoices.

**How to Update Profile**:

1. Profile icon (bottom right) > Edit Profile.
2. Make changes > Save. Some updates (e.g., phone change) require OTP verification.
3. For security: Changing email/phone triggers logout on other devices and requires re-verification.

**Deleting Your Account**:

- Profile > Account & Security > "Delete Account".
- Confirm with OTP/password.
- Process takes 30 days (during which you can cancel).
- All data (orders, preferences) permanently deleted; refunds/credits may be lost.
- Why delete? If switching apps or privacy concerns—contact support for data export first.

## 3. Security Incident Response Guide

QuickBite monitors for suspicious activity (e.g., logins from new devices/locations, unusual orders).

**Signs of Compromised Account**:

- Unrecognized logins/orders.
- Password change emails you didn't request.
- Failed login attempts notifications.
- Unauthorized address/payment changes.

**What to Do If You Suspect a Hack**:

1. **Immediately Change Password**: Log in (if possible) > Profile > Account & Security > Change Password.
2. **Enable/Reset 2FA**: If not enabled, turn it on. If compromised, disable old and set up new.
3. **Review Activity**: Check "Recent Activity" log for unfamiliar devices/locations > Log out all other sessions.
4. **Report to QuickBite**:
   - In-app: Profile > Help > "Account Security Issue" > Describe + screenshots.
   - Or email support@quickbite.com with account details.
5. **Additional Steps**:
   - Freeze linked payment methods via bank/app.
   - Scan device for malware.
   - Monitor for identity theft (e.g., unusual bank charges).

**QuickBite Actions on Report**:

- Temporary account lock if high risk.
- Investigation (usually <24 hours).
- If confirmed breach: Reset credentials, issue credits if fraudulent orders occurred, notify you.
- In severe cases: Permanent ban on abusive/fraudulent accounts.

## 4. Two-Factor Authentication (2FA) Setup and Management

2FA adds a second verification step (beyond password) for logins, password resets, high-value orders (>₹2000), or sensitive changes.

**Why Enable 2FA?**

- Prevents unauthorized access even if password is compromised.
- Recommended for all users, especially in high-fraud areas like Delhi.

**Setup Options**:

- **SMS OTP** (easiest): Code sent to registered phone.
- **Authenticator App** (most secure): Google Authenticator, Authy, Microsoft Authenticator (time-based codes).
- **Biometric** (fingerprint/face ID on supported devices): For quick logins after initial 2FA.

**Step-by-Step Enable 2FA**:

1. Profile > Account & Security > "Two-Factor Authentication".
2. Choose method (SMS or App).
3. For app: Scan QR code or enter setup key > Verify with first code.
4. Save 10 backup recovery codes (screenshot/store securely—never share!).
5. Confirm setup—future logins require second factor.

**Managing 2FA**:

- Change method: Disable old > Enable new (must be logged in).
- Lost authenticator: Use backup code or recovery process.
- Disable temporarily: Only for trusted scenarios (not recommended).

**Troubleshooting 2FA Issues**:

- Code not received? Check signal/spam, request resend, or switch to app method.
- Locked out? Use backup code or contact support (may require identity verification).
- New device? Re-setup 2FA after logging in.

**Security Best Practices**:

- Never share OTPs/codes (even with "support"—QuickBite never asks).
- Use strong, unique passwords + password manager.
- Enable login alerts (Profile > Notifications > Security Alerts).
- Log out on shared/public devices.
- Keep app updated for latest security patches.
